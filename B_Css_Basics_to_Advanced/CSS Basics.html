  • Built-in styel applied when no custom CSS defined.Provides basic readable layout.
  • The browser default style sheet,also known as the user agent style sheet is a built in stylesheet that each web browsers uses to apply default styles to a HTML element.
  • If no other Styles are defined by the web developer.
  • The purpose of this Trail is to ensure that web pages have some basic readable layout,even when no custom css have been applied.

    • How CSS Works?

  • Linking CSS to HTML :: CSS must be connected to the HTML document.
  • This can be done by three ways.

  • CSS Selectors :: CSS styles are applied to HTML element through what is called selectors, meaning we are going to select the pieces of HTML that we want to style.

  • CSS declaration :: This declaration is made up of a property and a value separated by a colon and enclosed in a declaration.
  • The property specifies what style or characteristics you want to change.

  • The cascade :: Cascade is one of the most important aspects of CSS.
  • It means that when multiple styles are applied to the same element, the browser follows certain rules.To determine which style takes precedence.

  • Inheritance :: IN CSS some styles like color, font, size, family and others are inherited by child elements from their parent element , while other properties like padding margin border are not.
  • This feature of CSS can be used to write more efficient code and maintain consistancy.

  • Rendering :: This is where we are going to see the actual effect of the CSS being appied to the element.
  • Once the breser has determined the appropriate styles for each element through cascading and inheritance, then it applies those styles and renders the web page for viewing.

    • How Browser Renders CSS?

  • Fetching Resources :: Before a browser can display the actual application or website, it needs to download the resources for the server
  • These resources can be of Javascript, CSS, HTML and any other files. so the process begins when the browser fetches the HTML file.

  • Parsing HTML :: The browser parses the HTML document into a document object modelwhich represents" DOM "
  • Which is a tree like Structure representing all the elements on the page.

  • Parsing CSS :: During the parsing of a HTML and CSS, the passing of CSS occurs simultaneously.
  • The browser parses any linked or embedded CSS into a CSS object model called CSS SOM or CSS ORM.
  • Which is a representation of all the styles that will be applied to the DOM.

  • Constructing Render Tree :: The browser combines the Dom and CSS samples into render tree.
  • this render tree contains information about each visible DOM element along with CSS properties that should be applied to the HTML element

  • Layout :: Once the render tree is built, then the browser calculates the layout, determine the exact position and size of each element on the screen.
  • The result of this process is a box model, which we will talk about it as we move on.It includes details like dimensions,padding,border and margin.

  • Painting :: The browser paint each node in the render tree to the screen following the order of the tree.
  • Painting involves applying the exact styles defined in the CSS.
  • Examples colors, gradients, images, borders, shadows, and so much more
